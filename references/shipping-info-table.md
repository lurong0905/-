# 发货信息搜集表

合同文件夹内的 `发货信息搜集表.xlsx` 是发货资料生成 workflow 的小型权威输入。首次处理新合同时可以由脚本自动预填，用户核对后，后续批次优先读取该表，避免反复读取大 PDF 合同。

## 文件位置和读取规则

- 文件名固定为 `发货信息搜集表.xlsx`，放在对应合同文件夹根目录。
- 如果文件存在，先读取该表；只有表内缺少必填字段、用户要求重新预填，或需要核验合同条款时才读取 PDF。
- 如果文件不存在，运行 `scripts/create_shipping_info_table.py <合同文件夹>` 生成预填表。
- 生成最终文件前，所有必确认字段必须来自信息表或用户明确回复。

## 工作表结构

### 合同固定信息

字段按三列维护：`字段`、`值`、`来源/说明`。建议字段如下：

| 字段 | 用途 |
| --- | --- |
| 客户体系 | 中核、国核、广核或其他 |
| 合同名称 | 发货通知、唛头、装箱清单的合同名称 |
| 合同号 | 发货通知合同号、唛头买方合同号 |
| 买方名称 | 买方/收货方 |
| 卖方名称 | 发货单位或供应商名称 |
| 工厂代码 | 中核默认收货地址匹配用 |
| 收货单位 | 通常为买方或合同指定收货单位 |
| 收货地址 | 到货地点、唛头到货地点 |
| 收货人 | 发货通知收货人 |
| 收货电话 | 唛头接货人电话 |
| 收货邮箱 | 需要邮件通知时使用 |
| 合同管理联系人 | 中核发货通知的商务合同处合同经理 |
| 项目管理联系人 | 只记录合同信息，不自动填采购申请处室项目负责人 |
| 合同要求交货时间 | 模板中的合同要求交货时间 |
| 发货地点 | 模板中的发货地点 |
| 发货单位 | 通常为卖方名称 |
| 发货单位联系人 | 发货通知发货单位联系人 |
| 发货单位联系电话 | 发货通知联系电话 |
| 立项单号默认值 | 中核模板通常填 `/` |
| 采购申请处室项目负责人默认值 | 中核模板通常填 `/` |

### 批次发货信息

每一行对应一个发货批次。建议字段如下：

| 字段 | 用途 |
| --- | --- |
| 批次 | 第一批、第二批等 |
| 发货清单文件 | 本批次仓库发货清单文件名 |
| 计划发运时间 | 必确认字段，不得推断 |
| 预计到达时间 | 资料不明确时必须确认 |
| 运单号/快递单号 | 未发运时可留空或填待补 |
| 运输方式 | 物流、快递、自送、专车等 |
| 装卸货方式 | 必确认字段；可多选 |
| 储存级别 | 必确认字段 |
| 储存条件 | 必确认字段：室内、室外或其它 |
| 是否生成最终版 | 是/否；必确认字段不全时只能生成待确认版 |
| 备注 | 其他批次说明 |

### 填写说明与选项

记录必填提醒和允许值，便于用户填表：

- `装卸货方式`：吊车、叉车、人工搬运、其它；允许多选，多个值用 `；` 分隔。
- `储存条件`：室内、室外、其它；选择其它时补充具体条件。
- `计划发运时间`：必须由用户填写或确认。
- `预计到达时间`：若合同或物流安排未明确，必须由用户填写或确认。

## 信息优先级

生成资料时按以下优先级取值：

1. 用户本次明确输入。
2. `发货信息搜集表.xlsx`。
3. 仓库装箱清单或发货清单。
4. Skill 内置客户默认信息。
5. PDF 合同首次预填结果。

冲突时以高优先级为准，并在最终说明中标明发生冲突的字段。
